# UI_DESIGN.md

# User Interface Design Rules

## Purpose

This document defines the visual design system, user experience principles, and UI standards for the project.

Every page, component, and visual element must follow these rules.

Before creating any UI, read this document completely.

---

# Design Philosophy

The interface must be:

- Clean
- Modern
- Simple
- Professional
- Consistent
- User-friendly

The design should focus on usability, not unnecessary decoration.

---

# Design Style

The project follows a modern SaaS-style design approach.

Characteristics:

- Clean layouts
- Clear typography
- Balanced spacing
- Minimal visual noise
- Smooth interactions
- Professional appearance

---

# Color System

Use a consistent color palette.

Do not randomly choose colors.

The project should define:

## Primary Color

Used for:

- Main buttons
- Important actions
- Brand elements


## Secondary Color

Used for:

- Supporting elements
- Highlights


## Neutral Colors

Used for:

- Backgrounds
- Borders
- Text
- Cards


## Status Colors

Use standard meanings:

Success:

Green

Warning:

Yellow/Orange

Error:

Red

Info:

Blue

---

# Color Usage Rules

Avoid using too many colors.

Maximum:

- 1 primary color
- 1 secondary color
- Neutral colors
- Status colors

Maintain visual consistency.

---

# Typography

Typography must be readable and consistent.

Use:

- Clear font hierarchy
- Proper line height
- Comfortable spacing

---

# Font Rules

Use modern readable fonts.

Recommended:

English:

Inter

Roboto

System UI


For Persian:

Vazirmatn

IRANSansX

Use only one main font family.

---

# Text Hierarchy

Every page should have:

## Heading

Large and clear.

## Subheading

Supporting information.

## Body Text

Readable content.

## Small Text

Secondary information.

---

# Spacing System

Use consistent spacing.

Avoid random margins and padding.

Preferred spacing scale:

```
4px
8px
12px
16px
24px
32px
48px
64px
```

---

# Layout Rules

Pages should use:

- Clear sections
- Proper alignment
- Enough whitespace

Avoid crowded interfaces.

---

# Container Rules

Content should have maximum width.

Avoid full-width content without reason.

Recommended:

```
max-width container
centered layout
```

---

# Card Design

Cards should be:

- Simple
- Clean
- Consistent

Cards may include:

- Rounded corners
- Soft shadows
- Clear padding

Avoid excessive shadows.

---

# Border Radius

Use consistent rounded corners.

Recommended:

Small:

8px

Medium:

12px

Large:

16px

---

# Buttons

Buttons must be consistent.

Every button should have:

- Clear purpose
- Good contrast
- Hover state
- Disabled state
- Loading state when needed

---

# Button Types

Primary Button:

Main action.

Example:

Save

Submit


Secondary Button:

Alternative action.


Danger Button:

Destructive actions.

Example:

Delete

---

# Forms Design

Forms must be:

- Simple
- Clear
- Easy to complete

Inputs must include:

- Label
- Placeholder when useful
- Validation message
- Focus state

---

# Navigation

Navigation must be:

- Simple
- Predictable
- Easy to understand

Users should always know:

Where they are.

Where they can go.

---

# Responsive Design

The UI must support:

Mobile

Tablet

Desktop

Design mobile-first.

---

# Dark Mode

If dark mode is implemented:

It must support:

- Proper contrast
- Readable text
- Correct colors
- Consistent components

Do not simply invert colors.

---

# Animations

Animations should improve user experience.

Use:

- Smooth transitions
- Loading animations
- Micro interactions

Avoid:

- Excessive movement
- Distracting effects

---

# Loading States

Every async action must have feedback.

Examples:

- Spinner
- Skeleton loading
- Disabled button

Never leave users wondering.

---

# Empty States

When there is no data:

Show:

- Helpful message
- Explanation
- Possible action

Example:

"No products found. Create your first product."

---

# Error States

Errors should be:

- Clear
- Friendly
- Actionable

Avoid technical messages.

---

# Icons

Use consistent icon styles.

Do not mix multiple icon libraries unnecessarily.

Icons should:

- Support meaning
- Improve understanding
- Not replace important text

---

# Tables

Tables should support:

- Clear headers
- Proper spacing
- Responsive behavior
- Pagination for large data

---

# Accessibility

Every UI must consider accessibility.

Rules:

- Proper semantic HTML
- Keyboard support
- Visible focus states
- Readable contrast
- Accessible forms

---

# UI Component Consistency

Before creating a new component:

Check if a similar component already exists.

Reuse:

- Buttons
- Inputs
- Cards
- Modals
- Tables

---

# Design Review

Before finalizing a UI:

Check:

- Is it consistent?
- Is it responsive?
- Is it easy to use?
- Is spacing correct?
- Are colors consistent?

---

# Final UI Rule

The interface must always feel:

Professional

Simple

Modern

Consistent

Easy to use
